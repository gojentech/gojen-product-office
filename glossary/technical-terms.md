## Purpose

Define shared technical terminology used in architecture, API, and engineering documentation.

## Terms

| Term | Definition |
| --- | --- |
| **API** | Application Programming Interface — contract for system-to-system interaction. |
| **ADR** | Architecture Decision Record — durable record of a significant technical decision. |
| **Architecture** | Structure of system components, responsibilities, and interfaces. |
| **Latency** | Time taken for a request or operation to complete. |
| **Availability** | Proportion of time a system is operable when expected. |
| **Idempotency** | Property where repeating the same request yields the same result without unintended side effects. |
| **SLA / SLO / SLI** | Service Level Agreement / Objective / Indicator — commitment, target, and measured signal. |
| **AuthN / AuthZ** | Authentication (who you are) / Authorization (what you may do). |
| **Event-driven** | Design style where state changes are communicated via events. |
| **Schema** | Formal structure of data or messages. |
| **Migration** | Controlled change to data structures or runtime systems. |
| **Rollback** | Reversion to a prior known-good release or state. |
| **Observability** | Ability to infer system state from logs, metrics, and traces. |
| **CI / CD** | Continuous Integration / Continuous Delivery (or Deployment). |
| **NFR** | Non-Functional Requirement — quality attribute such as performance or security. |
