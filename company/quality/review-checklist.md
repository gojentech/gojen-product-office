## Purpose

Provide reviewers a practical checklist for GAIOS and Product Office documents.

## Checklist

- [ ] Document Information table complete
- [ ] Version History updated
- [ ] Purpose is explicit in one short paragraph
- [ ] Scope in/out is clear
- [ ] No silent conflict with authoritative lifecycle docs
- [ ] Mermaid diagrams (if any) render conceptually and labels are short
- [ ] Tables have headers
- [ ] References point to real paths
- [ ] Approval table filled for Approved docs
- [ ] Change log entry present

## Review Outcomes

| Outcome | Meaning |
| --- | --- |
| Approve | Ready for Approver |
| Approve with comments | Non-blocking fixes tracked |
| Request changes | Blocking issues must be fixed |
| Reject | Wrong artifact or unsafe content |
