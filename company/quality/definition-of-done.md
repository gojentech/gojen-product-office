## Purpose

Define done for documentation and for sprint documentation work under GAIOS.

## Documentation Done

A document is done when:

1. All applicable quality gates pass.
2. Status and approvals match reality.
3. Consumers can navigate to related authoritative artifacts.
4. Open questions have owners and dates.

## Sprint Documentation Done

A sprint's documentation work is done when:

1. Planned artifacts exist in the correct folders.
2. GAIOS summaries updated if operating posture changed.
3. Decisions and risks filed where required.
4. Review checklist completed by a reviewer.

## Not Done Examples

- Draft left without owner
- Duplicate PRD content forked into GAIOS without links
- Approved stamp without approver name/date
