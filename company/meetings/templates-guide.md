## Canonical Template (do not edit)

Use the repository meeting template as the structural source of truth:

- [../../templates/meeting-template.md](../../templates/meeting-template.md)

This guide **references** that file. Authors must **not** modify, overwrite, or delete it when creating new agendas or minutes.

## How to Create a New Agenda or Minutes Doc

1. Copy the section structure from [meeting-template.md](../../templates/meeting-template.md) into a **new** markdown file under `company/meetings/` or the appropriate product meeting folder.  
2. Add GAIOS metadata (Document ID, Version 1.0.0, Status, Owner, Reviewer, Approver, dates, Purpose, Scope, Related Documents).  
3. Add the standard navigation block, including Back to START-HERE.  
4. Fill every section with real content for the meeting; do not leave placeholder or lorem text.  
5. Link dashboards, roadmaps, and decision drafts instead of pasting large duplicates.  
6. Register the meeting in [meeting-index.md](./meeting-index.md).

## Founder Board Specific Guidance

| Template concern | GAIOS practice |
| --- | --- |
| Attendees | Always include the three founders; note absences |
| Agenda | Prefer the shape in [Founder Board Cadence](./founder-board-cadence.md) |
| Decisions | Point to drafts in founder workspaces before the meeting |
| Actions | Transfer to each founder’s `action-items.md` the same day |
| Pre-reads | List the five role dashboards explicitly |
