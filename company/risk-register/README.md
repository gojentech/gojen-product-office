## Required Fields

Every risk document must include:

| Field | Description |
| --- | --- |
| Risk ID | Stable identifier (e.g., RISK-GPO-001) |
| Category | Knowledge, AI, Strategy, Process, Compliance, etc. |
| Probability | Low / Medium / High |
| Impact | Low / Medium / High |
| Mitigation | Controls and actions |
| Owner | Accountable role |
| Current Status | Open / Monitoring / Mitigating / Closed |

## Scoring Guide

| Level | Probability | Impact |
| --- | --- | --- |
| Low | Unlikely in next 6 months | Contained inconvenience |
| Medium | Plausible without controls | Material delay or rework |
| High | Likely without controls | Strategic or trust damage |
