## Purpose

Define when and how products or candidates move to archive status.

## Archive Triggers

| Trigger | Example |
| --- | --- |
| Strategic kill | Founder Board ends investment |
| Rejected intake | Candidate fails evaluation |
| Merge / replace | Product superseded by another offering |
| Long hold expiry | Hold window elapsed without revival |

## Archive Checklist

1. Record final status and rationale.
2. Mark GAIOS summary docs `Archived` in status when applicable.
3. Point to last known authoritative lifecycle path.
4. Capture lessons learned (minimum five bullets).
5. Update portfolio index inventory.
6. Ensure no active sprint plans assume the archived product.

## Retention

| Artifact class | Retention posture |
| --- | --- |
| Decision records | Retain indefinitely |
| Research summaries | Retain indefinitely |
| Working drafts | Retain if they explain decisions; otherwise note disposal |
| Secrets / credentials | Never store in archive docs |

## Revival

Revival requires a new intake evaluation. Historical IDs remain archived; new work gets new IDs and a fresh workspace plan.
