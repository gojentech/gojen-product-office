## Purpose

Define how GAIOS and GAIOS documents are versioned.

## Semver Rules

| Change type | Version bump | Example |
| --- | --- | --- |
| Compatible doc improvements | Patch | 1.0.0 → 1.0.1 |
| New domains or material OS capability | Minor | 1.0.0 → 1.1.0 |
| Incompatible restructure of OS meaning | Major | 1.x → 2.0.0 |

## Document vs System Versions

| Layer | Version field |
| --- | --- |
| Individual markdown document | Document Information Version |
| GAIOS operating system release | versions/ release docs |

A document may be 1.0.0 while GAIOS is 1.1.0 after new domains are added; release notes must explain the relationship.
