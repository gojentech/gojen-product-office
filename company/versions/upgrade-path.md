## Purpose

Explain how operators move from GAIOS v1.0 to later versions without breaking authority.

## Upgrade Principles

1. Prefer additive folders and documents.
2. Avoid rewriting authoritative lifecycle content during OS upgrades.
3. Publish release notes before declaring a new GAIOS version Approved.
4. Run link-health and authority-drift audits after upgrades.

## Rollback Posture

Documentation rollback means retaining prior Approved versions in history tables and clearly marking superseded docs — not deleting history.
